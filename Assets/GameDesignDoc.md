## 1. ゲーム概要

**ジャンル:** パズドラ風 落ち物パズル（タイムアタック制）  
**エンジン:** Unity (C#)  
**画面:** 960×540（16:9 横長）  
**入力:** マウス（PC）＋タッチ（スマホ）両対応（`Pointer.current` 使用）  
**フォント:** NotoSansJP-Bold SDF

---

## 2. ゲームフロー

```
起動 → タイトル画面 → モード選択(Classic/Action) → ゲームプレイ → TIME UP → リトライ → タイトル画面
```

### 2.1 タイトル画面
- ゲーム起動時に全画面オーバーレイで表示
- "漢字合体パズル" のタイトルテキスト
- 2つのモード選択ボタン:
  - **1マス入替モード** (Classic) — 緑色ボタン
  - **自由移動モード** (Action) — 紫色ボタン
- 各ボタンの下にモード別ハイスコアを表示
- 下部にモード操作説明テキスト

### 2.2 ゲームプレイ
- モード選択後、タイトルパネルが非表示になりゲーム開始
- 制限時間（初期60秒）のカウントダウン
- 合体成功で時間回復（+3秒）
- 手詰まり（合体可能な組み合わせゼロ）時は自動シャッフルが発動（手動シャッフルは不可）

### 2.3 ゲームオーバー
- 残り時間が0になると「TIME UP!」パネル表示
- 最終スコアとハイスコア更新を表示
- 「リトライ」ボタンでタイトル画面に戻る

## 3. ゲームモード

### 3.1 Classic モード（1マス入替モード）

| 項目 | 仕様 |
|------|------|
| 操作 | ピースをタップ/クリック → スワイプ方向に1マスだけ隣接ピースと入替え |
| 判定方法 | ドラッグ開始位置と終了位置の差分ベクトルで上下左右を決定 |
| 最低移動距離 | `TotalCellSize × 0.3` 以上のスワイプで方向確定 |
| 入替え後 | 連鎖判定を自動実行 |

### 3.2 Action モード（自由移動モード）

| 項目 | 仕様 |
|------|------|
| 操作 | ピースを掴んで盤面内を自由にドラッグ（パズドラ風） |
| 制限時間 | 1回のドラッグにつき5秒 |
| 押し退け | ドラッグ中のピースが他セルに侵入すると、元のピースが**スムーズアニメーション（0.12秒 EaseOutQuad）**で押し退けられる |
| ドラッグ中の見た目 | 半透明（α=0.85）、1.2倍スケール、sortingOrder=100（最前面） |
| ドラッグ終了後 | ピースが正しいグリッド位置にスナップし、連鎖判定を自動実行 |

## 6. マッチ判定ルール

操作終了後、以下の優先順位で盤面を判定する:

### 6.1 隣接レシピマッチ（最優先）
- 隣接する2ピースがレシピに一致 → 素材消去 → 結果漢字をスポーン

### 6.2 同一漢字3つ揃え（横・縦）
- 同じ漢字が縦または横に3つ並んでいる場合、消去して特殊結果漢字を生成

| 素材×3 | 結果 |
|--------|------|
| 木×3 | 森 |
| 日×3 | 晶 |
| 火×3 | 焱 |
| 人×3 | 众 |
| 山×3 | 嵐 |
| 石×3 | 磊 |

### 6.3 終端漢字の同種ペア消去
- 「どのレシピの素材にもならない漢字（終端漢字）」が隣接2つ並ぶと消去（300点）
- 終端漢字はレシピの結果にのみ登場し、素材としては登場しない漢字

## 7. 連鎖（コンボ）システム

```
操作終了 → マッチ判定 → 消去＋合体 → 重力落下 → 補充 → 再判定 → (ループ)
```

- 連鎖するごとにコンボカウントが+1
- 2コンボ以上で画面中央に「N Combo!」テキスト（虹色＋スケールバウンスアニメ）
- コンボボーナス: 各マッチのスコアに `(コンボ数-1) × 50` を加算

### 7.1 重力落下
- ピース消去後、空きセルの上にあるピースが下方向に落下
- 列ごとに下から詰める方式

### 7.2 補充
- 落下後の最上段の空きセルに、kanjiPoolからランダムな新ピースが上からアニメーション（0.3秒）で降ってくる

---

## 8. 自動シャッフル

- 連鎖終了後に「手詰まり判定」を実行
- 現在の盤面でマッチが存在せず、かつどの隣接ペアを入れ替えてもマッチが成立しない場合 → **手詰まり**
- 手詰まり時は**ノーペナルティで自動シャッフル**（盤面全消去→新規ランダム補充）
- 「No More Moves! Shuffle!」テキストを水色＋スケールバウンス（0.8秒）で画面中央に表示
- シャッフル後にもマッチ判定→連鎖を実行
- シャッフル後の偶発連鎖はスコア＋時間ボーナスがそのまま付与される（ラッキーボーナス）

## 11. ハイスコア

- `PlayerPrefs` で永続保存
- モード別に独立管理:
  - Classic: キー `HighScore_Classic`
  - Action: キー `HighScore_Action`
- タイトル画面: 両モードのハイスコアを表示
- ゲーム中: 選択モードのハイスコアを左上に表示
- ゲームオーバー時にハイスコア更新チェック（`NEW RECORD!` 表示）

## 17. セットアップ手順

Unity Editor メニュー `Tools > Setup Puzzle TD` を実行すると以下が自動生成される:

1. フォルダ構成の作成
2. KanjiRecipe アセット（35件）の再生成
3. KanjiPiecePrefab の作成（SpriteRenderer + TextMeshPro + BoxCollider2D）
4. シーンオブジェクトの構成:
   - Camera（正射影、サイズ5）
   - PuzzleSystem（PuzzleBoard + PuzzleManager）
   - Canvas + 全UI要素
   - EventSystem
   - ParticleSystem
   - GameManager（全参照の自動紐付け）
